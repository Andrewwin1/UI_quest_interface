# Quest Control System - Система управления квест-румом

Веб-интерфейс для управления квест-румом на базе Raspberry Pi + Arduino Mega по RS485.

## Архитектура

- **Arduino Mega** - управление платой расширения (15 PWM, 15 выходов, 15 входов, 15 аналоговых)
- **Raspberry Pi** - веб-сервер, база данных, движок сценариев
- **RS485** - связь между RPi и Arduino (протокол Queen Bus)

## Возможности

✅ **Панель управления** - ручное управление всеми устройствами в реальном времени
✅ **Конструктор сценариев** - визуальное создание сценариев с действиями
✅ **База данных** - хранение устройств, звуков и сценариев
✅ **Звуковое сопровождение** - загрузка и воспроизведение звуков
✅ **WebSocket** - обновление состояния в реальном времени
✅ **Настройка устройств** - добавление и редактирование устройств

## Установка на Raspberry Pi

### 1. Установка зависимостей

```bash
cd /home/admin
git clone <ваш репозиторий> GarryHome
cd GarryHome/rpi_server

# Создать виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### 2. Настройка RS485

Убедитесь, что RS485 адаптер подключен к GPIO UART:
- По умолчанию используется `/dev/serial0`
- Скорость: 115200 baud
- ID устройства Arduino: 16

Проверьте наличие порта:
```bash
ls -l /dev/serial*
```

### 3. Создание папки для звуков

```bash
mkdir -p sounds
```

### 4. Запуск сервера

```bash
python main.py
```

Сервер запустится на `http://0.0.0.0:8000`

### 5. Автозапуск (systemd)

Создайте файл `/etc/systemd/system/quest.service`:

```ini
[Unit]
Description=Quest Control System
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/home/admin/GarryHome/rpi_server
ExecStart=/home/admin/GarryHome/rpi_server/venv/bin/python main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

Включите автозапуск:
```bash
sudo systemctl enable quest.service
sudo systemctl start quest.service
sudo systemctl status quest.service
```

## Использование

### Открыть веб-интерфейс

Откройте браузер и перейдите на:
```
http://<IP_адрес_RPi>:8000
```

### Добавление устройств

1. Перейдите на вкладку **Устройства**
2. Нажмите **+ Добавить устройство**
3. Заполните:
   - Название (например: "Дверной замок")
   - Тип (output/pwm/input/analog)
   - Пин (0-14)
   - Иконка (эмодзи)
   - Группа (опционально)
4. Сохраните

### Создание сценария

1. Перейдите на вкладку **Сценарии**
2. Нажмите **+ Новый**
3. Введите название и описание
4. Добавьте действия:
   - **⏱️ Задержка** - пауза в миллисекундах
   - **🔌 Включить выход** - управление цифровым выходом
   - **💡 Установить PWM** - управление яркостью/мощностью
   - **🔊 Воспроизвести звук** - запуск звукового файла
   - **⏳ Ждать сигнал** - ожидание сигнала от входа
   - **🔁 Повторить N раз** - цикл действий
5. Сохраните сценарий
6. Запустите кнопкой **▶️**

### Загрузка звуков

1. Перейдите на вкладку **Звуки**
2. Нажмите **📤 Загрузить звук**
3. Выберите аудиофайл (MP3, WAV, OGG)
4. Звук появится в списке и будет доступен в сценариях

## Типы действий в сценариях

| Действие | Описание | Параметры |
|----------|----------|-----------|
| ⏱️ Задержка | Пауза перед следующим действием | Миллисекунды |
| 🔌 Выход | Включить/выключить цифровой выход | Пин, состояние (0/1) |
| 💡 PWM | Установить мощность PWM | Пин, мощность (0-255), плавность |
| 🔊 Звук | Воспроизвести звуковой файл | Файл, громкость (0-100) |
| 🔇 Остановить звук | Остановить воспроизведение | - |
| ⏳ Ждать сигнал | Ждать сигнала от входа | Пин, состояние, таймаут |
| 🔁 Повторить | Повторить блок действий N раз | Количество |
| ♾️ Цикл | Бесконечный цикл до остановки | - |

## API Endpoints

### Устройства
- `GET /api/devices` - список устройств
- `POST /api/devices` - создать устройство
- `PUT /api/devices/{id}` - обновить устройство
- `DELETE /api/devices/{id}` - удалить устройство

### Управление
- `POST /api/control/output` - управление выходом
- `POST /api/control/pwm` - управление PWM
- `GET /api/state` - текущее состояние

### Сценарии
- `GET /api/scenarios` - список сценариев
- `POST /api/scenarios` - создать сценарий
- `PUT /api/scenarios/{id}` - обновить сценарий
- `DELETE /api/scenarios/{id}` - удалить сценарий
- `POST /api/scenarios/{id}/start` - запустить сценарий
- `POST /api/scenarios/{id}/stop` - остановить сценарий

### Звуки
- `GET /api/sounds` - список звуков
- `POST /api/sounds/upload` - загрузить звук
- `DELETE /api/sounds/{id}` - удалить звук
- `POST /api/sounds/{id}/play` - воспроизвести звук
- `POST /api/sounds/stop` - остановить все звуки

### WebSocket
- `WS /ws` - real-time обновления состояния

## Отладка

### Проверка подключения к Arduino

```python
from queen_driver import QueenBusDevice

queen = QueenBusDevice(port='/dev/serial0', baudrate=115200, device_id=16)
if queen.connect():
    print("Подключено!")
    queen.start_polling(0.1)
    # Проверка
    queen.set_output(0, 1)  # Включить выход 0
    print(f"Входы: {queen.inputs}")
    print(f"Аналог: {queen.analog}")
else:
    print("Ошибка подключения")
```

### Логи сервера

```bash
# Запуск с логами
python main.py

# Логи systemd
sudo journalctl -u quest.service -f
```
